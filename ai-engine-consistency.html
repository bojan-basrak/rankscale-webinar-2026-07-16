Rankscale variance study · De'Longhi (US)

How consistent are AI engines when they recommend brands?

If you ask the same question twice, do you get the same brands, in the same order? We measured the run-to-run stability of nine AI engines across 1,722 real executions — and where the wobble actually lives.

7 topics × 9 engines = 63 cells 1,722 executions ~46 days tracked Source: Rankscale REST API

Key findings

The short version

Consistency is a headline score for "same answer twice?"; three sub-metrics say in what way an engine varies.

The headline

No AI engine is highly consistent — and the churn is in the lineup, not the ranking

Across every engine, the exact set of brands named shifts run-to-run (Lineup Stability ~54%), while the order of the brands that do appear is far steadier (Order Stability ~76%) and the top pick holds ~71% of the time. Being "the answer" once doesn't mean you're the answer next time.

API vs GUI

Structured APIs beat live-web surfaces

The four [API] engines occupy the top of the consistency leaderboard; the live-web [GUI] surfaces (Google AI Overview, ChatGPT) sit at the bottom.

Stabilization

Estimates settle by ~10–15 runs

Consistency scores wobble for the first few executions, then flatten. ~25 runs per prompt is comfortably enough to trust the number.

Range

Best to worst: 35.8% → 22.0%

Claude 4.5 Haiku [API] leads; Google AI Overview [GUI] trails — a 14-point spread in overall Consistency.

How we measure it

One headline, three diagnoses

The unit is a single prompt run repeatedly on a single engine. We take each execution's answer, extract the ordered list of brands it names (matched against Rankscale's own tracked brand set), and compare those lists across runs. One headline number, three that explain it.

Method & caveats

How to read these numbers responsibly

Read with these in mind
  • Brands seen through Rankscale. Extraction is limited to Rankscale's tracked brand set per prompt; brands it doesn't track are invisible, which biases all scores slightly upward.
  • Real-world drift is included. Over ~46 days the market itself shifts, so a little "inconsistency" is the world changing, not the model. Isolating pure model stochasticity would need dense same-day repeat runs.
  • Consistency (RBO) is a comparative scale. Top-weighted overlap on short lists runs low in absolute terms — use it to rank engines, not as a "70% = good" gauge.
  • Extraction is deterministic string-matching against Rankscale's alias lists (first-mention order = position), not a language model — fast and reproducible, with minor matching noise.
